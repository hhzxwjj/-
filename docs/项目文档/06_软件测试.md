# 六、软件测试

## 6.1 测试策略

### 6.1.1 测试目标

确保书法报名管理系统在功能、性能、安全等方面满足需求规格说明，保证系统稳定可靠运行。

### 6.1.2 测试范围

| 测试类型 | 测试对象 | 优先级 |
|----------|----------|--------|
| **功能测试** | 用户认证、课程管理、报名缴费、签到课时、消息通信 | 高 |
| **接口测试** | 所有 HTTP API 和 WebSocket 事件 | 高 |
| **安全测试** | 密码加密、SQL注入、XSS、权限控制、频率限制 | 高 |
| **兼容性测试** | Chrome、Edge、Firefox 浏览器 | 中 |
| **性能测试** | 页面响应时间、并发处理能力 | 中 |

### 6.1.3 测试环境

| 配置项 | 环境参数 |
|--------|----------|
| 操作系统 | Windows 11 |
| Python | 3.10.11 |
| Flask | 2.0.1 |
| 浏览器 | Chrome 120 / Edge 120 / Firefox 121 |
| 测试工具 | Postman 10.20、curl、Chrome DevTools |
| 网络 | 本地局域网（延迟 < 1ms） |

### 6.1.4 测试准入准出标准

| 标准类型 | 条件 |
|----------|------|
| **准入条件** | 1. 所有功能开发完成并通过自测<br>2. 数据库初始化脚本可正常运行<br>3. 测试环境搭建完毕 |
| **准出条件** | 1. 严重/高危缺陷修复率 100%<br>2. 中低优先级缺陷修复率 ≥ 80%<br>3. 测试用例执行率 100%，通过率 ≥ 90% |

### 6.4.4 测试结论

1. **功能完整性**：系统覆盖了需求规格中定义的全部核心功能，包括用户认证、课程管理、报名缴费、签到课时、消息通信等模块。
2. **安全性**：密码采用 PBKDF2 加密，SQL 注入通过参数化查询有效防护，权限控制严格区分管理员和普通用户。
3. **稳定性**：经过 38 个测试用例验证，未发现严重或高危缺陷，系统可稳定运行。
4. **建议**：
   - 生产环境建议将 SQLite 替换为 PostgreSQL 以支持更高并发
   - 建议接入真实短信服务和支付接口
   - 建议补充自动化单元测试（pytest）覆盖核心业务逻辑
