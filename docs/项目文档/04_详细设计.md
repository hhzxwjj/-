```
│                                     │

交互逻辑：
1. 用户点击"立即报名" → POST /enroll/{course_id}（含备选课程数组）
```

```
交互逻辑：
1. 选择日期 → 页面加载该课程所有 paid 状态学员

   - 旧状态 present，新状态非 present → attendance_count - 1
5. 提交成功后通过 WebSocket 广播 attendance_update 事件
```

## 4.2 模块详细分解

### 4.2.1 用户认证模块（M1）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `login()` | POST: username/password/captcha 或 phone/verification_code | 登录成功跳转 / 错误提示页 | 1. 判断登录模式<br>2. 验证码模式：校验验证码→查手机号→创建用户（新用户）→写入Session<br>3. 密码模式：校验图形验证码→查用户名→verify_password→写入Session<br>4. 检查信息完整性，不完整则跳转完善页 |
| `register()` | POST: phone/verification_code/name/child_name | 注册成功跳转 / 错误提示 | 1. 校验验证码<br>2. 查重手机号<br>3. 生成随机username和password<br>4. hash_password入库<br>5. 自动登录，跳转成功页 |
| `complete_profile()` | POST: name/child_name/phone | 更新成功跳转首页 / 错误提示 | 1. 校验长度和手机号格式<br>2. UPDATE users<br>3. 设置 session['profile_complete'] = True |
| `change_password()` | JSON: old_password/new_password | JSON: success/message | 1. 校验原密码<br>2. 新密码长度≥6<br>3. hash_password更新 |
| `logout()` | - | 跳转登录页 | 清除全部 Session |
| `check_profile_completeness()` | - | None / Redirect | before_request 中间件：排除白名单路由→检查session缓存→无缓存则查库→标记完整性状态→不完整则强制跳转 |

### 4.2.2 课程管理模块（M2）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `courses()` | - | 渲染 courses.html（课程列表） | SELECT * FROM courses |
| `course_detail(course_id)` | URL参数: course_id | 渲染 course_detail.html | 1. 管理员：查所有课程<br>2. 普通用户：查未满课程<br>3. 管理员额外查报名学生列表和备选课程标记<br>4. 查当前用户报名状态 |
| `add_course()` | POST: name/description/total_hours/price/start_time/end_time/max_students | 重定向 admin | 1. 输入校验（非空、正数、长度）<br>2. INSERT INTO courses |
| `admin()` | - | 渲染 admin.html | SELECT * FROM courses |

### 4.2.3 报名缴费模块（M3）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `enroll(course_id)` | POST JSON: alternate_courses[] | JSON: success/message | 1. 检查登录和信息完整性<br>2. 查课程余量<br>3. 查重复报名<br>4. 保存 course_id 和 alternate_courses 到 session |
| `payment()` | - | JSON: success/message | 1. 从 session 取 course_id<br>2. 再次校验课程余量和重复报名<br>3. BEGIN TRANSACTION<br>4. INSERT enrollments (paid)<br>5. INSERT alternate_courses<br>6. UPDATE courses.current_students + 1<br>7. COMMIT / ROLLBACK<br>8. 清除 session 课程信息 |

### 4.2.4 签到课时模块（M4）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `attendance(course_id)` | GET: course_id<br>POST: class_date + attendance_{enrollment_id} | 渲染 attendance.html / 重定向 admin | GET: 1. 查课程信息<br>2. 计算本次课次<br>3. 查所有 paid 学员<br>POST: 1. 遍历学员<br>2. 查/插/更新 attendances<br>3. 根据状态变化更新 attendance_count |
| `hours_management()` | - | 渲染 hours_management.html | 1. 查所有课程<br>2. 对每个课程查学生及 attendance_count<br>3. 计算平均课时<br>4. 标记 below_average 和 hours_gap |
| `temp_adjust()` | POST JSON: enrollment_id/course_id/class_date | JSON: success/message | 1. 校验报名记录和课程存在<br>2. 计算课次<br>3. BEGIN TRANSACTION<br>4. INSERT attendances (present)<br>5. UPDATE enrollments attendance_count + 1<br>6. COMMIT<br>7. WebSocket 广播 temp_adjust_success |

### 4.2.5 学员管理模块（M5）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `move_student()` | POST JSON: student_id/current_course_id/target_course_id | JSON: success/message | 1. 校验目标课程余量<br>2. 查当前报名记录<br>3. 查目标课程是否已报名<br>4. BEGIN TRANSACTION<br>5. UPDATE courses 人数-1/+1<br>6. UPDATE enrollments course_id<br>7. COMMIT |
| `remove_student()` | POST JSON: enrollment_id | JSON: success/message | 1. 查报名记录<br>2. BEGIN TRANSACTION<br>3. UPDATE courses.current_students - 1<br>4. DELETE attendances<br>5. DELETE enrollments<br>6. COMMIT |
| `add_student()` | POST JSON: course_id/child_name/parent_name/phone | JSON: success/message | 1. 查课程存在和余量<br>2. 生成随机 username/password<br>3. BEGIN TRANSACTION<br>4. INSERT users<br>5. INSERT enrollments (pending)<br>6. UPDATE courses.current_students + 1<br>7. COMMIT |

### 4.2.6 消息通信模块（M6）

| 函数/接口 | 输入 | 输出 | 处理逻辑 |
|-----------|------|------|----------|
| `messages()` | - | 渲染 messages.html | 1. 管理员：查所有 user 角色用户<br>2. 普通用户：查管理员信息 |
| `api_conversation(other_user_id)` | URL参数: other_user_id | JSON: messages[] | 1. 权限校验（普通用户只能查管理员）<br>2. UPDATE 对方消息为 read<br>3. SELECT 双方聊天记录 ORDER BY created_at |
| `send_message()` | POST: receiver_id/content | JSON: success/message | 1. 校验 receiver_id 和内容长度<br>2. INSERT messages<br>3. 返回成功 |
| `handle_connect()` | - | emit 'connected' | 1. 校验登录<br>2. join_room(user_{id})<br>3. admin角色额外 join_room('admin') |
| `handle_send_message(data)` | WebSocket: {message, room} | emit 'new_message' to room | 1. 校验登录和内容<br>2. 房间名白名单校验<br>3. broadcast 到指定 room |

## 4.4 数据安全设计

### 4.4.1 密码加密

| 项目 | 设计内容 |
|------|----------|
| **算法** | PBKDF2-HMAC-SHA256 |
| **盐值** | secrets.token_hex(16)，每次加密随机生成 |
| **迭代次数** | 100,000 次 |
| **存储格式** | `salt$derived_key_hex` |
| **兼容性** | 支持旧版纯 SHA256 哈希的无感升级验证 |
| **比对方式** | `secrets.compare_digest()` 防止时序攻击 |

### 4.4.2 权限控制

| 控制点 | 实现方式 | 说明 |
|--------|----------|------|
| **角色区分** | Session 中存储 `role` 字段 | admin / user |
| **管理员路由保护** | 每个管理员路由首行检查 `session['role'] == 'admin'` | 未授权返回 403 或重定向 |
| **普通用户隔离** | 课程详情中普通用户只能看到未满课程 | 管理员可查看全部 |
| **消息权限** | 普通用户只能与管理员通信 | api_conversation 中强制校验 |
| **信息完整性强制** | `check_profile_completeness` 中间件 | 未完善信息禁止访问核心功能 |

### 4.4.3 SQL注入防护

| 措施 | 实现方式 |
|------|----------|
| **参数化查询** | 所有 SQL 均使用 `?` 占位符，禁止字符串拼接 |
| **输入长度限制** | 用户名≤50、姓名≤50、密码≥6、消息≤500 |
| **格式白名单** | 手机号正则 `^1[3-9]\d{9}$`；房间名只允许 `course_{id}` / `user_{id}` |
| **ORM-like封装** | 使用 `sqlite3.Row` 结果集，避免原始索引访问 |

### 4.4.4 敏感数据脱敏

| 数据项 | 展示方式 | 示例 |
|--------|----------|------|
| **密码** | 数据库仅存储哈希，任何场景不返回明文 | - |
| **手机号** | 前端展示脱敏 `138****8888`（可扩展） | 完整号仅用于登录校验 |
| **验证码** | 仅存在于 Session，5分钟过期，使用后立即清除 | - |
| **Session ID** | Flask 自动管理，Cookie 设置 HttpOnly | 防止 XSS 窃取 |

### 4.4.5 其他安全措施

| 措施 | 实现 |
|------|------|
| **请求频率限制** | `rate_limit` 装饰器：IP 维度，60秒窗口内最多5次请求（登录/发验证码） |
| **图形验证码** | 4位数字验证码，300秒过期，防止自动化暴力破解 |
| **Session 安全** | `SESSION_COOKIE_HTTPONLY=True`，`SAMESITE='Lax'`，2小时过期 |
| **错误处理** | 404/500/429 统一处理，生产环境不暴露堆栈信息 |
| **日志记录** | 登录失败、异常操作记录到 `app.log`，便于审计 |
