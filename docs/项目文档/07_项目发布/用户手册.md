# 墨香书院书法报名管理系统——用户手册

## 一、系统简介

**墨香书院书法报名管理系统**是一款面向书法培训机构的在线课程管理与报名签到平台。系统支持学员在线浏览课程、提交报名意向、完成缴费，支持管理员进行课程发布、学员签到、课时统计和实时消息沟通。

### 1.1 主要功能

| 角色 | 功能 |
|------|------|
| **学员（普通用户）** | 浏览课程、在线报名、模拟缴费、查看考勤、消息沟通、修改密码 |
| **管理员** | 课程管理、学员签到、人员调整、课时统计、临时调课、消息回复、学员手动添加 |

### 1.2 技术架构

- **前端**：HTML5 + CSS3 + JavaScript（Jinja2 模板引擎）
- **后端**：Python 3.10+ / Flask 2.0.1
- **实时通信**：Flask-SocketIO 5.1.1
- **数据库**：SQLite（开发）/ MySQL（生产）
- **部署**：Docker 容器化 / 本地直接运行

### 1.3 运行环境

- 浏览器：Chrome 120+ / Edge 120+ / Firefox 121+
- 分辨率：推荐 1920×1080 及以上
- 网络：支持局域网或互联网访问

### 2.1 环境准备

确保已安装：
- Python 3.10 或更高版本
- pip 包管理器

### 2.2 安装依赖

```bash
cd d:/trae_projects/BaoMing
python -m venv .venv
```

## 三、普通用户操作指南

### 3.1 用户注册

**操作路径**：首页 → 右上角"注册" → 填写注册信息

**步骤**：
1. 在浏览器中打开系统首页 `http://localhost:5000`
2. 点击导航栏右侧的 **"注册"** 链接，进入注册页面
3. 输入 **手机号**（11位中国大陆手机号）
4. 点击 **"获取验证码"**（演示环境直接显示验证码）
5. 输入收到的 **6位验证码**
6. 填写 **家长姓名** 和 **孩子姓名**
7. 点击 **"注册"** 按钮

**注意事项**：
- 手机号格式必须正确（1开头，第二位3-9，共11位数字）
- 验证码5分钟内有效，过期需重新获取
- 每个手机号只能注册一次
- 注册成功后会自动分配随机用户名和密码，请妥善保存

### 3.2 用户登录

**操作路径**：首页 → 右上角"登录"

**步骤**：
1. 点击导航栏右侧的 **"登录"** 链接
2. 在登录页面，可选择两种登录方式：
   - **用户名密码登录**：输入用户名、密码、验证码图片中的4位数字
   - **手机验证码登录**：输入手机号和短信验证码（演示环境直接显示）
3. 点击 **"登录"** 按钮

**注意事项**：
- 验证码图片可点击刷新
- 连续5次登录失败后，系统会触发频率限制，需等待60秒后重试
- 登录后2小时内无操作，Session会自动过期，需重新登录

### 3.3 完善个人信息

**触发条件**：首次登录后，如果姓名或孩子姓名为"未知"，系统会强制跳转至完善资料页面

**步骤**：
1. 在弹出的完善资料页面，填写 **家长姓名**、**孩子姓名**、**手机号**
2. 点击 **"保存"** 按钮
3. 资料完善后即可正常使用全部功能

### 3.4 浏览课程

**操作路径**：首页 → "课程列表" 或 首页功能区 "立即报名"

**页面说明**：
- 课程卡片显示：课程名称、课时数、价格、起止时间、剩余名额
- 已满课程会显示"已满"状态，无法报名
- 点击课程卡片可查看课程详情

### 3.5 报名课程

**操作路径**：课程列表 → 点击课程卡片 → "立即报名"

**步骤**：
1. 在课程列表中选择心仪的课程，点击进入详情页
2. 阅读课程信息后，点击 **"立即报名"** 按钮
3. 系统提示"请先完成缴费"
4. 点击 **"去缴费"** 按钮
5. 在缴费页面确认课程信息和金额，点击 **"确认缴费"**
6. 缴费成功后，显示"报名成功"

**注意事项**：
- 每人每门课程只能报名一次
- 课程人数达到上限后无法报名
- 信息不完整的用户需先完善资料才能报名

### 3.6 查看个人考勤

**操作路径**：首页 → "用户中心" → 考勤记录

**页面说明**：
- 显示已报名课程的出勤记录
- 每次上课的出勤状态（出勤/缺席）
- 累计出勤次数

### 3.7 消息中心

**操作路径**：首页 → "消息中心"

**功能说明**：
- 普通用户只能与管理员进行消息沟通
- 支持实时在线聊天（WebSocket）
- 未读消息会有红点提示
- 历史聊天记录自动保存

**操作步骤**：
1. 进入消息中心，左侧显示管理员联系人
2. 点击联系人进入对话窗口
3. 在输入框中输入消息内容（最多500字）
4. 按回车或点击发送按钮发送消息
5. 管理员回复后，消息会实时显示在对话窗口中

### 3.8 修改密码

**操作路径**：用户中心 → "修改密码"

**步骤**：
1. 输入 **原密码**
2. 输入 **新密码**（不少于6位）
3. 确认新密码
4. 点击 **"保存"** 按钮

## 四、管理员操作指南

### 4.1 管理员登录

系统启动时会自动创建管理员账号：
- **用户名**：`admin`
- **密码**：优先从环境变量 `ADMIN_DEFAULT_PASSWORD` 读取，未设置则自动生成12位随机密码并打印在控制台

**登录步骤**：
1. 在登录页面输入用户名 `admin`
2. 输入管理员密码
3. 输入验证码
4. 点击登录

### 4.2 管理后台首页

**操作路径**：登录后 → 导航栏"管理员后台"

**页面功能**：
- 课程管理：查看所有课程列表
- 课时管理：查看各课程学员课时统计
- 快捷操作：查看详情、签到、删除课程

### 4.3 添加课程

**操作路径**：管理员后台 → "添加课程"折叠面板

**步骤**：
1. 点击 **"添加课程"** 区域的展开按钮
2. 填写课程信息：
   - **课程名称**（必填，不超过100字）
   - **课程描述**（必填）
   - **总课时**（必填，正整数）
   - **价格**（必填，正数，单位：元）
   - **开始时间**（必填）
   - **结束时间**（必填）
   - **最大人数**（必填，正整数）
3. 点击 **"添加课程"** 按钮
4. 课程添加成功后自动显示在课程列表中

**注意事项**：
- 开始时间必须早于结束时间
- 总课时、价格、最大人数必须为正数

### 4.4 删除课程

**操作路径**：管理员后台 → 课程列表 → "删除"按钮

**步骤**：
1. 在课程列表中找到要删除的课程
2. 点击该课程卡片上的 **"删除"** 按钮
3. 在弹出的确认对话框中点击 **"确定"**
4. 课程及其关联的报名记录、签到记录、备选课程记录将被一并删除

**警告**：删除操作不可恢复，请谨慎操作！

### 4.5 课程详情与学员管理

**操作路径**：管理员后台 → 课程列表 → "查看详情"

**页面功能**：
- 查看课程基本信息
- 查看已报名学员列表
- **调整人员**：将学员转移到其他有余量的课程
- **移除学员**：删除学员的报名记录
- **临时调课**：为学员添加补课签到记录

#### 4.5.1 调整学员课程

**步骤**：
1. 在学员列表中找到目标学员
2. 点击 **"调整"** 按钮
3. 在弹出的下拉框中选择 **目标课程**（仅显示有余量的课程）
4. 点击 **"确认调整"**
5. 系统会校验：目标课程是否存在且未满、学员是否在源课程中、学员是否已在目标课程中
6. 调整成功后，源课程人数-1，目标课程人数+1

#### 4.5.2 移除学员

**步骤**：
1. 在学员列表中找到目标学员
2. 点击 **"移除"** 按钮
3. 在确认对话框中点击 **"确定"**
4. 系统会删除该学员的报名记录和相关签到记录，课程人数-1

#### 4.5.3 临时调课

**步骤**：
1. 在学员列表中找到目标学员
2. 点击 **"临时调课"** 按钮
3. 选择 **补课日期**
4. 点击 **"确认"**
5. 系统会为该学员创建一条 present 状态的签到记录，并更新出勤次数

### 4.6 签到管理

**操作路径**：管理员后台 → 课程列表 → "签到"按钮

**步骤**：
1. 在课程列表中点击目标课程的 **"签到"** 按钮
2. 系统显示该课程的所有报名学员列表
3. 选择 **上课日期**
4. 为每位学员选择出勤状态：
   - **出勤（present）**：计入出勤次数
   - **缺席（absent）**：不计入出勤次数
5. 点击 **"提交签到"** 按钮
6. 系统自动统计并更新每位学员的出勤次数

**注意事项**：
- 可以修改历史日期的签到记录
- 状态从 absent 改为 present 时，attendance_count +1
- 状态从 present 改为 absent 时，attendance_count -1

### 4.7 课时统计

**操作路径**：管理员后台 → "课时管理" 标签

**页面说明**：
- 按课程分组显示所有学员的出勤次数
- 计算每门课程的平均出勤次数
- **红色标记**：出勤次数低于平均值的学员
- 显示每位学员与平均值的差距课时数

**用途**：帮助管理员快速发现缺课较多的学员，及时跟进。

### 4.8 消息管理

**操作路径**：首页 → "消息中心"

**管理员消息中心功能**：
- 查看所有学员联系人列表
- 联系人按置顶状态 + 最新消息时间排序
- 点击联系人右侧的 **"置顶"** 图标可将常用联系人置顶
- 支持实时消息收发

## 五、常见问题（FAQ）

### Q1：忘记密码怎么办？
**A**：当前版本暂不支持密码找回功能。请联系管理员重置密码。管理员可通过数据库直接修改 `users` 表的 `password` 字段（需使用 `hash_password` 函数生成新哈希）。

### Q2：为什么报名按钮是灰色的？
**A**：可能原因：① 课程已满（current_students ≥ max_students）；② 您已报名该课程；③ 您的个人信息不完整（姓名或孩子姓名为"未知"）。请先完善个人资料。

### Q3：缴费是真实的吗？
**A**：当前版本为演示环境，缴费流程为模拟支付，不涉及真实资金交易。生产环境需接入微信支付/支付宝等真实支付接口。

### Q4：如何切换数据库到 MySQL？
**A**：设置环境变量 `USE_MYSQL=true`，并配置 `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE`。详见《系统部署手册》。

### Q5：签到记录可以修改吗？
**A**：可以。管理员进入签到页面，选择历史日期，修改学员的出勤状态后重新提交即可。系统会自动重新计算 attendance_count。

### Q6：验证码过期了怎么办？
**A**：验证码有效期为5分钟（300秒）。如果过期，请刷新验证码图片重新输入。

### Q7：为什么登录时提示"请求过于频繁"？
**A**：系统设置了频率限制：同一IP在60秒内最多允许5次请求。请等待60秒后重试。

### Q8：WebSocket消息发送失败怎么办？
**A**：请检查：① 网络连接是否正常；② 是否已登录（未登录用户无法连接WebSocket）；③ 浏览器是否支持WebSocket（Chrome/Edge/Firefox均支持）。
