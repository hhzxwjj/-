## 二、部署概述

本系统支持三种部署方式：
1. **本地开发部署**：直接在 Windows/Linux 上运行 Python 应用
2. **Docker 容器化部署**：使用 Docker 构建镜像并运行容器
3. **生产环境部署**：MySQL 数据库 + Gunicorn/uWSGI + Nginx 反向代理

## 四、部署方式一：本地直接部署（Windows/Linux）

### 4.1 获取代码

```bash
# 克隆仓库
git clone https://github.com/hhzxwjj/BaoMing.git
cd BaoMing
```

### 4.2 创建虚拟环境

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/macOS
python3 -m venv .venv
source .venv/bin/activate
```

### 4.3 安装依赖

```bash
pip install -r requirements.txt
```

依赖列表：
- Flask==2.0.1
- Flask-SocketIO==5.1.1
- Pillow==9.0.1
- python-socketio==5.5.0
- python-engineio==4.3.0
- PyMySQL==1.0.2

### 4.4 配置环境变量（可选）

```bash
# Windows CMD
set SECRET_KEY=your-secret-key-here
set ADMIN_DEFAULT_PASSWORD=admin123
set FLASK_DEBUG=false
set FLASK_HOST=0.0.0.0
set FLASK_PORT=5000

# Windows PowerShell
$env:SECRET_KEY="your-secret-key-here"
$env:ADMIN_DEFAULT_PASSWORD="admin123"

# Linux/macOS Bash
export SECRET_KEY="your-secret-key-here"
export ADMIN_DEFAULT_PASSWORD="admin123"
export FLASK_DEBUG="false"
export FLASK_HOST="0.0.0.0"
export FLASK_PORT="5000"
```

### 4.5 启动应用

```bash
# 使用虚拟环境中的 Python
.venv/Scripts/python.exe run.py

# Linux
.venv/bin/python run.py
```

启动成功后输出：
```
[*] 启动服务: http://127.0.0.1:5000
[*] Debug模式: False
```

### 4.6 访问验证

在浏览器中打开 `http://localhost:5000`，应显示系统首页。

## 五、部署方式二：Docker 容器化部署

### 5.1 构建 Docker 镜像

项目根目录已包含 `Dockerfile`：

```dockerfile
# 书法报名管理系统 Docker 镜像
FROM python:3.10-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目代码
COPY app/ ./app/
COPY static/ ./static/
COPY templates/ ./templates/
COPY run.py .
COPY mysql_schema.sql .

# 暴露端口
EXPOSE 5000

# 环境变量（生产环境请覆盖）
ENV SECRET_KEY=""
ENV ADMIN_DEFAULT_PASSWORD=""
ENV USE_MYSQL="false"

# 启动命令
CMD ["python", "run.py"]
```

**构建命令**：

```bash
# 进入项目根目录
cd BaoMing

# 构建镜像
docker build -t calligraphy-system:v1.0 .

# 查看镜像
docker images | grep calligraphy-system
```

### 5.2 运行容器

```bash
# 方式一：简单运行（SQLite 模式，数据不持久化）
docker run -d \
  --name calligraphy-app \
  -p 5000:5000 \
  -e SECRET_KEY="your-secret-key" \
  -e ADMIN_DEFAULT_PASSWORD="admin123" \
  calligraphy-system:v1.0

# 方式二：带数据卷持久化（推荐）
docker run -d \
  --name calligraphy-app \
  -p 5000:5000 \
  -v $(pwd)/data:/app/data \
  -e SECRET_KEY="your-secret-key" \
  -e ADMIN_DEFAULT_PASSWORD="admin123" \
  calligraphy-system:v1.0
```

### 5.3 查看容器状态

```bash
# 查看运行中的容器
docker ps

# 查看容器日志
docker logs -f calligraphy-app

# 进入容器内部
docker exec -it calligraphy-app /bin/bash
```

### 5.4 停止与重启

```bash
# 停止容器
docker stop calligraphy-app

# 重启容器
docker restart calligraphy-app

# 删除容器
docker rm -f calligraphy-app
```

### 5.5 Docker Compose 部署（推荐）

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  app:
    build: .
    container_name: calligraphy-app
    ports:
      - "5000:5000"
    environment:
      - SECRET_KEY=${SECRET_KEY:-change-me-in-production}
      - ADMIN_DEFAULT_PASSWORD=${ADMIN_DEFAULT_PASSWORD:-admin123}
      - USE_MYSQL=${USE_MYSQL:-false}
      - FLASK_DEBUG=${FLASK_DEBUG:-false}
      - FLASK_HOST=0.0.0.0
      - FLASK_PORT=5000
    volumes:
      - ./instance:/app/instance
      - ./calligraphy_system.db:/app/calligraphy_system.db
    restart: unless-stopped

  # 如需 MySQL，取消下面注释
  # mysql:
  #   image: mysql:8.0
  #   container_name: calligraphy-mysql
  #   environment:
  #     - MYSQL_ROOT_PASSWORD=root
  #     - MYSQL_DATABASE=calligraphy_system
  #   ports:
  #     - "3306:3306"
  #   volumes:
  #     - mysql_data:/var/lib/mysql
  #   restart: unless-stopped

# volumes:
#   mysql_data:
```

**部署命令**：

```bash
# 启动服务

# 查看状态

# 查看日志
```

## 六、部署方式三：生产环境部署（MySQL + Nginx）

### 6.1 MySQL 数据库配置

#### 6.1.1 安装 MySQL

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install mysql-server

# CentOS/RHEL
sudo yum install mysql-server
sudo systemctl start mysqld
sudo systemctl enable mysqld
```

#### 6.1.2 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE calligraphy_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（可选）
CREATE USER 'calligraphy'@'%' IDENTIFIED BY 'your-password';
GRANT ALL PRIVILEGES ON calligraphy_system.* TO 'calligraphy'@'%';
FLUSH PRIVILEGES;
EXIT;
```

#### 6.1.3 导入表结构

```bash
mysql -u root -p calligraphy_system < mysql_schema.sql
```

#### 6.1.4 配置环境变量切换到 MySQL

```bash
export USE_MYSQL="true"
export MYSQL_HOST="localhost"
export MYSQL_PORT="3306"
export MYSQL_USER="calligraphy"
export MYSQL_PASSWORD="your-password"
export MYSQL_DATABASE="calligraphy_system"
```

### 6.2 Nginx 反向代理配置

创建 `/etc/nginx/sites-available/calligraphy`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 静态资源
    location /static/ {
        alias /var/www/calligraphy/static/;
        expires 30d;
    }

    # WebSocket 支持
    location /socket.io/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 其他请求
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/calligraphy /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 6.3 HTTPS 配置（SSL 证书）

使用 Let's Encrypt 免费证书：

```bash
# 安装 certbot
sudo apt-get install certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

配置 HTTPS 后，请将 Flask 中的 `SESSION_COOKIE_SECURE` 设为 `True`：

```python
app.config.update(
    SESSION_COOKIE_SECURE=True,  # HTTPS 环境下启用
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SAMESITE='Lax'
)
```

### 6.4 使用 systemd 管理服务

创建 `/etc/systemd/system/calligraphy.service`：

```ini
[Unit]
Description=Calligraphy Management System
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/calligraphy
Environment="PATH=/var/www/calligraphy/.venv/bin"
Environment="SECRET_KEY=your-secret-key"
Environment="ADMIN_DEFAULT_PASSWORD=admin123"
Environment="USE_MYSQL=true"
Environment="MYSQL_HOST=localhost"
Environment="MYSQL_PORT=3306"
Environment="MYSQL_USER=calligraphy"
Environment="MYSQL_PASSWORD=your-password"
Environment="MYSQL_DATABASE=calligraphy_system"
Environment="FLASK_DEBUG=false"
Environment="FLASK_HOST=127.0.0.1"
Environment="FLASK_PORT=5000"
ExecStart=/var/www/calligraphy/.venv/bin/python run.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable calligraphy
sudo systemctl start calligraphy
sudo systemctl status calligraphy
```

## 十、常见问题排查

### Q1：启动时提示 `SECRET_KEY` 未设置
**解决**：设置环境变量 `SECRET_KEY`，或修改 `app/__init__.py` 中的默认密钥。

### Q2：Windows 下中文日志乱码
**解决**：在 `run.py` 启动前添加：
```python
import sys
sys.stdout.reconfigure(encoding='utf-8')
```
或在 PowerShell 中执行：`chcp 65001`

### Q3：MySQL 连接失败
**解决**：检查 `USE_MYSQL=true` 时，是否已安装 `pymysql`；确认 MySQL 服务已启动；检查用户名密码。

### Q4：Docker 容器无法访问
**解决**：检查端口映射 `-p 5000:5000`；确认防火墙未阻止 5000 端口。

### Q5：WebSocket 连接失败
**解决**：确保 Nginx 配置中包含 WebSocket upgrade 头；检查浏览器是否支持 WebSocket。

### Q6：静态资源加载失败
**解决**：生产环境建议使用 Nginx 直接托管 static 目录；检查 `static/` 目录权限。
