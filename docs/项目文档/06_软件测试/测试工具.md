**`conftest.py` 中的关键 fixture**：

## 四、Flask 测试客户端

### 4.1 工具简介

Flask 内置的 `test_client` 是一个 WSGI 应用包装器，可以在不启动真实服务器的情况下模拟 HTTP 请求。本项目所有集成测试均基于 Flask 测试客户端构建。

### 4.2 使用示例

```python
from app import app

with app.test_client() as client:
    # GET 请求
    resp = client.get('/courses')
    assert resp.status_code == 200
    
    # POST 请求（表单）
    resp = client.post('/login', data={
        'username': 'admin',
        'password': 'admin123',
        'captcha': '1234'
    })
    
    # POST 请求（JSON）
    resp = client.post('/change_password',
        data=json.dumps({'old_password': 'old', 'new_password': 'new123'}),
        content_type='application/json')
    data = json.loads(resp.data)
    assert data['success'] is True
    
    # Session 操作
    with client.session_transaction() as sess:
        sess['captcha'] = '1234'
        sess['captcha_created_at'] = time.time()
```

### 4.3 在测试中的应用

本项目使用 Flask 测试客户端完成了以下验证：
- **路由映射**：验证所有 `@app.route` 装饰器是否正确映射 URL
- **请求处理**：验证 GET/POST 请求的参数解析和响应生成
- **Session 管理**：验证登录状态、验证码存储、用户信息缓存
- **重定向**：验证权限拦截后的 302 重定向行为
- **错误处理**：验证 404/429/500 错误处理器的响应格式

## 五、Flask-SocketIO 测试客户端

### 5.1 工具简介

Flask-SocketIO 提供了 `test_client` 类，用于在测试环境中模拟 WebSocket 连接和事件收发，无需启动真实的 Socket.IO 服务器。

### 5.2 使用示例

```python
from app import app, socketio

with app.test_client() as http_client:
    # 创建 SocketIO 测试客户端（关联 HTTP 客户端的 Session）
    sio_client = socketio.test_client(app, flask_test_client=http_client)
    
    # 验证连接成功
    assert sio_client.is_connected()
    
    # 接收连接欢迎消息
    received = sio_client.get_received()
    assert received[0]['name'] == 'connected'
    
    # 发送事件
    sio_client.emit('join_course', {'course_id': 1})
    
    # 接收响应
    received = sio_client.get_received()
    event_names = [r['name'] for r in received]
    assert 'joined_course' in event_names
    
    sio_client.disconnect()
```

### 5.3 测试覆盖的 WebSocket 事件

| 事件名称 | 测试内容 | 对应 pytest 用例 |
|----------|----------|------------------|
| `connect` | 已登录用户连接成功，未登录被拒绝 | `test_websocket_connect_admin` / `test_websocket_connect_unauthenticated` |
| `disconnect` | 断开连接日志记录 | 集成在连接测试中 |
| `join_course` / `leave_course` | 加入/离开课程房间 | `test_websocket_join_leave_course` |
| `send_message` | 消息广播到指定房间 | `test_websocket_send_message` |
| `attendance_update` | 管理员签到更新广播 | `test_websocket_attendance_update_requires_admin` |

## 七、curl 命令行工具

### 7.1 工具简介

curl 是一个命令行工具，用于通过 URL 语法在命令行下工作，支持多种协议（HTTP、HTTPS、FTP 等）。适合快速验证接口可用性。

### 7.2 常用命令示例

```bash
# 获取验证码图片
curl -o captcha.png http://localhost:5000/captcha

# 登录（保存 Cookie）
curl -X POST http://localhost:5000/login \
  -d "username=admin&password=admin123&captcha=1234" \
  -c cookie.txt

# 获取课程列表（使用 Cookie）
curl http://localhost:5000/courses -b cookie.txt

# 报名课程
curl -X POST http://localhost:5000/enroll/1 \
  -H "Content-Type: application/json" \
  -d '{"alternate_courses":[]}' \
  -b cookie.txt

# 发送消息
curl -X POST http://localhost:5000/send_message \
  -d "receiver_id=1&content=Hello" \
  -b cookie.txt

# 获取 API 联系人列表
curl http://localhost:5000/api/contacts -b cookie.txt
```

## 九、DB Browser for SQLite

### 9.1 工具简介

DB Browser for SQLite（原 SQLite Database Browser）是一款开源的可视化 SQLite 数据库管理工具，支持浏览表数据、执行 SQL、导入导出。

### 9.2 本项目使用场景

| 场景 | 操作 |
|------|------|
| 验证数据写入 | 报名缴费后查看 `enrollments` 表是否插入记录，`courses.current_students` 是否 +1 |
| 验证事务一致性 | 删除课程后检查 `enrollments`、`attendances`、`alternate_courses` 关联记录是否被清理 |
| 验证密码哈希 | 查看 `users.password` 字段是否为 `salt$hash` 格式 |
| 验证签到统计 | 对比 `attendances` 表中 `status='present'` 的数量与 `enrollments.attendance_count` 是否一致 |
| 数据完整性检查 | 使用 "Browse Data" 查看外键约束是否生效 |

### 9.3 常用 SQL 验证语句

```sql
-- 检查密码哈希格式
SELECT username, password FROM users WHERE role = 'admin';

-- 验证报名后课程人数更新
SELECT name, current_students, max_students FROM courses WHERE id = 1;

-- 验证签到次数统计
SELECT e.id, e.attendance_count, COUNT(a.id) as actual_count
FROM enrollments e
LEFT JOIN attendances a ON a.enrollment_id = e.id AND a.status = 'present'
GROUP BY e.id;

-- 验证消息未读状态
SELECT COUNT(*) FROM messages WHERE status = 'unread';

-- 验证置顶记录
SELECT * FROM chat_pins;
```
