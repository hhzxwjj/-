# 二、需求分析与建模

## 2.1 项目意义与价值

### 用户价值
- **家长/学员**：随时随地通过手机浏览课程、在线报名缴费，实时查看孩子课时进度，与机构即时沟通
- **书法培训机构**：告别纸质档案，实现学员信息、缴费、课时的一站式数字化管理
- **管理员**：通过数据面板快速掌握各课程报名情况、学员出勤率，支持科学排课

### 业务价值
- 降低人工录入错误率，提升报名转化率
- 课时数据透明化，减少家长与机构间的纠纷
- 签到数据沉淀，为后续课程优化提供数据支撑

### 技术价值
- 实践 Flask 全栈开发、SQLite 关系型数据库设计、WebSocket 实时通信
- 涵盖完整的用户认证、权限控制、事务处理、安全防护等工程实践

## 2.2 用户角色

| 角色 | 描述 | 核心权限 |
|------|------|----------|
| **管理员（Admin）** | 培训机构运营人员 | 课程CRUD、学员管理、签到打卡、课时查看、消息回复、人员调整 |
| **普通用户（User）** | 学员家长 | 浏览课程、报名缴费、查看报名记录、完善个人信息、与管理员私信 |
| **访客（Visitor）** | 未登录用户 | 仅可查看登录/注册页面，需注册登录后才能使用核心功能 |

## 2.3 功能需求

### 功能需求列表

| 编号 | 功能模块 | 需求描述 | 优先级 |
|------|----------|----------|--------|
| FR-01 | 用户注册 | 家长通过手机号+验证码注册，系统自动生成账号密码 | 高 |
| FR-02 | 用户登录 | 支持用户名密码登录和手机号验证码登录两种方式 | 高 |
| FR-03 | 个人信息管理 | 用户完善/修改姓名、孩子姓名、手机号 | 高 |
| FR-04 | 课程浏览 | 展示课程列表（名称、价格、课时、时间、余量） | 高 |
| FR-05 | 课程详情 | 查看课程详细信息，显示已报名学生列表（管理员可见） | 高 |
| FR-06 | 在线报名 | 选择课程并提交报名，支持选择备选课程 | 高 |
| FR-07 | 模拟缴费 | 模拟支付流程，缴费成功后创建报名记录 | 高 |
| FR-08 | 课程管理 | 管理员发布、查看课程，管理报名学生 | 高 |
| FR-09 | 签到管理 | 管理员按课程、按日期为学员打卡（出勤/缺勤/请假） | 中 |
| FR-10 | 课时统计 | 自动统计学员已上课时，标记低于平均课时的学员 | 中 |
| FR-11 | 临时调课 | 管理员为学员添加临时签到记录，支持跨课程调课 | 中 |
| FR-12 | 消息中心 | 家长与管理员通过 WebSocket 实时发送/接收消息 | 中 |
| FR-13 | 人员调整 | 管理员将学员从A课程调整至B课程 | 低 |
| FR-14 | 验证码安全 | 登录/注册时验证图形验证码，防止暴力破解 | 高 |
| FR-15 | 请求频率限制 | 对登录等敏感接口进行IP级频率限制 | 高 |

## 2.4 非功能需求

| 类别 | 需求项 | 具体指标 |
|------|--------|----------|
| **性能** | 页面响应时间 | 普通页面 < 500ms，数据库查询 < 200ms |
| **性能** | 并发支持 | 演示环境支持 20 人同时在线 |
| **安全** | 密码存储 | PBKDF2-HMAC-SHA256 + 随机盐值，迭代10万次 |
| **安全** | 会话安全 | Cookie 设置 HttpOnly、SameSite=Lax，2小时过期 |
| **安全** | 输入校验 | 所有用户输入进行长度限制和格式校验，防止SQL注入 |
| **安全** | 权限控制 | 基于 Session 的角色鉴权，未授权访问自动跳转 |
| **可用性** | 浏览器兼容 | 支持 Chrome、Edge、Firefox 最新版 |
| **可用性** | 易用性 | 界面中文展示，关键操作有提示，表单有校验反馈 |
| **可靠性** | 数据一致性 | 报名、缴费、调课等操作使用数据库事务 |
| **可维护性** | 代码规范 | 遵循 PEP8，关键函数有 docstring 注释 |

### 核心用例规约（以"报名课程"为例）

| 用例名称 | 报名课程 |
|----------|----------|
| **参与者** | 家长/学员 |
| **前置条件** | 用户已登录，个人信息已完善 |
| **后置条件** | 系统创建报名记录，课程当前人数+1 |
| **主事件流** | 1. 用户浏览课程列表 → 2. 进入课程详情 → 3. 点击"立即报名" → 4. 选择备选课程 → 5. 确认报名 → 6. 进入缴费页面 → 7. 模拟支付成功 → 8. 系统创建报名记录 |
| **异常流** | E1：课程已满 → 提示"课程名额已满"<br>E2：已报名该课程 → 提示"请勿重复报名"<br>E3：信息不完整 → 跳转完善信息页面 |

### 关系类型说明

| 关系 | 类型 | 说明 |
|------|------|------|
| users → enrollments | 1:N | 一个用户可报名多个课程 |
| courses → enrollments | 1:N | 一个课程可被多个用户报名 |
| enrollments → attendances | 1:N | 一条报名记录对应多次签到 |
| enrollments → alternate_courses | 1:N | 一条报名可设置多个备选课程 |
| users → messages (sender) | 1:N | 一个用户可发送多条消息 |
| users → messages (receiver) | 1:N | 一个用户可接收多条消息 |
