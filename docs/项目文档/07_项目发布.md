# 七、项目发布

## 7.1 用户手册

### 7.1.1 系统简介

**系统名称**：书法培训班报名管理系统（Calligraphy Training Enrollment System）

**功能概述**：
本系统面向书法培训机构及其学员家长，提供在线课程浏览、报名缴费、课时管理、签到统计、消息沟通等一站式数字化管理服务。系统支持双模式登录（用户名密码/手机号验证码），区分管理员和家长两种角色，满足不同用户群体的使用需求。

**适用对象**：
- 书法培训机构管理员（课程发布、学员管理、签到打卡、课时统计）
- 学员家长（课程浏览、在线报名、查看课时、与机构沟通）

### 7.1.2 安装与部署

#### 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 或 Linux |
| Python | 3.10 及以上版本 |
| 内存 | 2GB 及以上 |
| 硬盘 | 100MB 可用空间 |
| 浏览器 | Chrome / Edge / Firefox 最新版 |

#### 安装步骤

**步骤1：获取项目代码**

```bash
git clone https://gitee.com/your-repo/BaoMing.git
cd BaoMing
```

**步骤2：创建虚拟环境（推荐）**

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate
```

**步骤3：安装依赖**

```bash
pip install -r requirements.txt
```

依赖清单：
- Flask==2.0.1
- Flask-SocketIO==5.1.1
- Pillow==9.0.1
- python-socketio==5.5.0
- python-engineio==4.3.0

**步骤4：启动系统**

```bash
python run.py
```

启动成功后将看到服务运行在 http://0.0.0.0:5000/

默认管理员账号：用户名 `admin`，密码见控制台输出（首次启动自动生成随机密码，可通过环境变量 `ADMIN_DEFAULT_PASSWORD` 配置）

### 7.1.3 操作指南

#### 家长/学员操作

**1. 注册账号**

| 步骤 | 操作 |
|------|------|
| 1 | 打开系统首页，点击"立即注册" |
| 2 | 输入手机号，点击"获取验证码"（演示环境直接显示） |
| 3 | 输入验证码、家长姓名、孩子姓名 |
| 4 | 点击"注册"，成功后保存系统生成的用户名和密码 |

**2. 登录系统**

支持密码登录（用户名+密码+图形验证码）或验证码登录（手机号+短信验证码）。

**3. 浏览与报名课程**

进入课程列表 → 点击课程查看详情 → 选择备选课程（可选）→ 点击"立即报名"→ 模拟缴费 → 报名成功。

**4. 查看课时与消息沟通**

用户中心查看报名记录；消息中心与管理员在线沟通，消息实时送达。

#### 管理员操作

**1. 发布课程**

管理后台填写课程名称、描述、课时、价格、时间、名额 → 添加课程。

**2. 签到管理**

选择课程 → 进入签到页面 → 选择日期 → 为每位学员标记出勤/缺勤/请假 → 提交签到。

**3. 学员管理**

课程详情页可查看学员列表、调整人员（跨课程转移）、移除学员；也可手动添加新学员。

**4. 课时统计与临时调课**

课时管理页面查看各课程学员课时，低于平均课时的学员标红预警；支持临时添加调课签到记录。

### 7.1.4 常见问题（FAQ）

**Q1：忘记密码怎么办？**
> 演示环境可联系管理员重置密码。生产环境建议通过手机号验证码重置。

**Q2：注册时提示"手机号已注册"怎么办？**
> 说明该手机号已存在账号，请使用验证码登录方式直接登录。

**Q3：报名后如何缴费？**
> 演示环境自动模拟支付成功。真实环境需集成微信支付/支付宝。

**Q4：签到后发现状态选错了怎么办？**
> 重新选择该日期，修改学员状态后再次提交，系统会自动重新计算课时。

**Q5：消息发送失败怎么办？**
> 请检查网络连接，刷新页面后重试。WebSocket 连接异常时会自动尝试重连。

## 7.2 系统部署

### 7.2.1 本地部署（开发/演示环境）

执行 `python run.py` 即可在本地 5000 端口启动服务。

### 7.2.2 服务器部署（生产环境建议）

**方案：Gunicorn + Nginx**

**步骤1：安装 Gunicorn**

```bash
pip install gunicorn eventlet
```

**步骤2：使用 Gunicorn 启动**

```bash
gunicorn -k eventlet -w 1 -b 127.0.0.1:8000 run:app
```

> 注意：Flask-SocketIO 需要使用 eventlet worker，且多 worker 模式下需要消息队列支持，单 worker 模式最简单。

**步骤3：配置 Nginx 反向代理**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /static {
        alias /path/to/BaoMing/static;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**步骤4：使用 systemd 守护进程（Linux）**

```ini
# /etc/systemd/system/calligraphy.service
[Unit]
Description=Calligraphy Enrollment System
After=network.target

[Service]
User=www-data
WorkingDirectory=/path/to/BaoMing
ExecStart=/path/to/BaoMing/.venv/bin/gunicorn -k eventlet -w 1 -b 127.0.0.1:8000 run:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl enable calligraphy
sudo systemctl start calligraphy
```
