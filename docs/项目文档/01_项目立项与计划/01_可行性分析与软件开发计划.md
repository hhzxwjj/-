## 1.2 选题背景与动机

### 背景
随着传统文化教育的普及，少儿及成人书法培训班数量快速增长。传统报名方式依赖线下登记、纸质表格和人工统计，存在以下痛点：
- **报名效率低**：家长需现场排队填写纸质报名表，高峰期拥堵严重
- **信息管理混乱**：学员信息、缴费记录、课时统计分散在不同表格中，查询困难
- **沟通成本高**：调课、请假等通知依赖微信群，消息容易被淹没
- **数据安全隐患**：纸质档案易丢失，敏感信息无加密保护

### 目标
开发一套面向书法培训机构的在线报名管理系统，实现：
1. 在线课程浏览与报名缴费
2. 学员信息与课时数字化管理
3. 签到统计与临时调课支持
4. 站内消息沟通（家长↔管理员）
5. 管理员后台一站式运营

## 1.3 可行性分析

### 1.3.1 技术可行性

| 维度 | 分析 | 结论 |
|------|------|------|
| **技术栈成熟度** | 采用 Python Flask（Web框架成熟稳定）、SQLite（轻量数据库）、SocketIO（实时通信），均为业界主流技术 | 可行 |
| **团队能力** | 团队具备 Python Web 开发、数据库设计、前端基础开发能力 | 可行 |
| **第三方依赖** | 仅依赖 Pillow（图像处理）生成验证码，无复杂外部 API 依赖 | 可行 |
| **部署难度** | 单进程 Flask 应用，可在 Windows/Linux 一键启动，适合课程演示环境 | 可行 |

### 1.3.2 经济可行性

| 成本项 | 金额（预估） | 说明 |
|--------|-------------|------|
| 服务器/域名 | ¥0（演示阶段） | 本地或学校内网部署 |
| 开发工具 | ¥0 | VS Code、Git、Draw.io 均为免费工具 |
| 短信服务 | ¥0（演示阶段） | 验证码采用后端模拟生成方式 |
| **总成本** | **≈ ¥0** | 教育演示项目，无商业运营成本 |

### 1.3.3 操作可行性

- **用户意愿**：家长普遍习惯使用手机完成报名缴费，系统提供手机号验证码登录，操作门槛低
- **操作简便性**：界面采用传统表单+表格布局，符合中老年管理员使用习惯
- **培训成本**：管理员仅需掌握课程发布、签到打卡两个核心功能，30分钟即可上手

## 1.5 版本迭代计划

### 实际版本发布记录（基于飞书项目管理工具跟踪）

| 版本号 | 发布日期 | 状态 | 变更摘要 |
|--------|----------|------|----------|
| **v1.0** | 2026/04/11 | 已发布 | 基础功能：用户管理、课程管理、签到管理、消息通知 |
| **v1.4** | 2026/04/15 | 已发布 | 安全增强：密码哈希升级（PBKDF2）、Session安全配置、输入验证强化 |
| **v1.5** | 2026/05/07 | 已发布 | Bug修复与体验优化：登录修复、折叠组件、验证码显示优化 |

### V1.0 —— 核心功能版（第1-2周）
- 用户注册/登录（用户名密码 + 手机号验证码双模式）
- 课程列表展示与详情查看
- 在线报名与模拟缴费
- 管理员课程发布与学生管理
- 数据库基础设计与初始化

### V2.0 —— 运营增强版（第3周）
- 签到管理（按课程、按日期打卡）
- 课时统计与缺课预警
- 临时调课功能
- 消息中心（WebSocket 实时通信）
- 验证码安全机制与请求频率限制

### V3.0 —— 稳定优化版（第4周）
- 代码安全加固（密码加密、SQL注入防护、XSS防御）
- 测试数据生成脚本
- 异常处理与日志记录完善
- 用户手册与部署文档编写
- 性能优化与 Bug 修复
