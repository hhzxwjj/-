## 5.3 代码规范

### 5.3.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| **模块/包名** | 全小写，下划线分隔 | `database.py`, `main.py` |
| **类名** | 大驼峰 | `DatabaseConnection` |
| **函数/方法** | 全小写，下划线分隔 | `hash_password()`, `get_db_connection()` |
| **常量** | 全大写，下划线分隔 | `PHONE_PATTERN` |
| **变量** | 全小写，下划线分隔 | `db_path`, `user_id` |
| **模板文件** | 全小写，下划线分隔 | `login.html`, `user_center.html` |
| **路由** | 全小写，斜杠分隔 | `/login`, `/admin/attendance/<id>` |

### 5.3.2 注释规范

| 场景 | 规范 | 示例 |
|------|------|------|
| **模块注释** | 文件顶部三引号文档字符串 | `"""生成测试数据：6个课程，约30名学员"""` |
| **函数注释** | 函数名下三引号说明 | `def hash_password(password):\n    """密码加密函数（使用pbkdf2）"""` |
| **路由注释** | `# 路由：`前缀 | `# 路由：登录` |
| **WebSocket注释** | `# WebSocket事件：`前缀 | `# WebSocket事件：处理客户端连接` |
| **关键逻辑** | 行内注释说明意图 | `# 验证码使用后立即失效` |

### 5.3.4 安全检查清单

| 检查项 | 实现状态 | 位置 |
|--------|----------|------|
| 密码使用哈希存储 | ✅ | `hash_password()`, `verify_password()` |
| 所有SQL使用参数化查询 | ✅ | 所有 `cursor.execute("... ? ...", params)` |

## 5.4 Git使用

### 5.4.1 提交规范（Conventional Commits）

本项目遵循 Conventional Commits 规范，提交信息格式如下：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**常用 type：**

| type | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具配置 |

**提交示例：**

```
feat(login): 增加手机号验证码登录功能

- 集成短信验证码接口（模拟）
- 增加验证码校验逻辑
- 支持新用户自动注册

Closes #12
```

### 5.4.2 分支管理策略

| 分支 | 用途 | 保护策略 |
|------|------|----------|
| `main` | 主分支，稳定版本 | 禁止直接推送，需PR合并 |
| `dev` | 开发分支，集成测试 | 需Code Review后合并 |
| `feature/*` | 功能开发分支 | 从dev切出，开发完成后合并回dev |
| `hotfix/*` | 紧急修复分支 | 从main切出，修复后合并回main和dev |
