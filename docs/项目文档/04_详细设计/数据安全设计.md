### 2.1 HTTPS（生产环境）

| 环境 | 协议 | 说明 |

| 开发环境 | HTTP | `SESSION_COOKIE_SECURE=False`，本地开发使用 |

```nginx
    }
    
    location / {
        proxy_pass http://127.0.0.1:5000;

    }
}
```

### 2.2 Session Cookie 安全属性

```python
app.config.update(
    SESSION_COOKIE_SECURE=False,      # 生产环境改为 True
    SESSION_COOKIE_HTTPONLY=True,     # 禁止 JavaScript 访问
    SESSION_COOKIE_SAMESITE='Lax',    # 跨站请求时 Cookie 发送策略
    PERMANENT_SESSION_LIFETIME=timedelta(hours=2)  # 2小时过期
)
```

| 属性 | 值 | 安全作用 |
|------|-----|----------|
| `Secure` | True（生产） | 仅通过 HTTPS 传输 |
| `HttpOnly` | True | 防止 XSS 窃取 Session |
| `SameSite` | Lax | 防止 CSRF（第三方 POST 请求不携带 Cookie） |
| `Max-Age` | 7200秒 | 限制 Session 有效时间 |

## 3. 认证安全设计

### 3.1 密码安全策略

#### 3.1.1 密码哈希算法（PBKDF2-HMAC-SHA256）

```python
def hash_password(password: str) -> str:
    """密码哈希，返回 salt$hash 格式"""
    salt = secrets.token_hex(16)          # 256位随机盐
    key = hashlib.pbkdf2_hmac(
        'sha256',                         # 哈希算法
        password.encode('utf-8'),         # 密码明文
        salt.encode('utf-8'),             # 盐值
        100000                            # 迭代次数（10万次）
    )
    return f'{salt}${key.hex()}'          # 存储格式

def verify_password(password: str, hashed_password: str) -> bool:
    """验证密码，兼容旧版 SHA256"""
    if '$' in hashed_password:
        # 新版 PBKDF2 格式
        salt, key = hashed_password.split('$')
        new_key = hashlib.pbkdf2_hmac(
            'sha256',
            password.encode('utf-8'),
            salt.encode('utf-8'),
            100000
        )
        return secrets.compare_digest(new_key.hex(), key)
    else:
        # 兼容旧版纯 SHA256
        old_hash = hashlib.sha256(password.encode('utf-8')).hexdigest()
        return secrets.compare_digest(old_hash, hashed_password)
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 算法 | PBKDF2-HMAC-SHA256 | NIST 推荐标准 |
| 盐长度 | 256 bit (32 byte hex) | 足够随机，防彩虹表 |
| 迭代次数 | 100,000 | 抵抗暴力破解 |
| 输出格式 | `salt$hash` | 便于存储和解析 |
| 比对方式 | `secrets.compare_digest()` | 防时序攻击 |

#### 3.1.2 密码升级机制

```
系统启动时（init_admin）：
  查询管理员记录
    │
    ├── 不存在 → 创建新管理员（使用 PBKDF2）
    │
    └── 存在 → 尝试用默认密码验证
              │
              ├── 验证失败 → 更新为 PBKDF2 格式
              │
              └── 验证通过 → 不做修改
```

### 3.2 验证码安全机制

#### 3.2.1 图形验证码

| 属性 | 值 | 说明 |
|------|-----|------|
| 长度 | 4 位 | 数字字符（2-9，排除易混淆的 0/1） |
| 有效期 | 300 秒（5分钟） | 超过时间自动失效 |
| 存储位置 | Flask Session | 服务端存储，防客户端篡改 |
| 图片干扰 | 干扰线 + 噪点 | 防止 OCR 自动识别 |
| 使用后销毁 | 立即清除 | 防止验证码复用 |

```python
def generate_captcha(return_base64=True):
    chars = '23456789'  # 排除 0 和 1，减少误识别
    captcha_text = ''.join(secrets.choice(chars) for _ in range(4))
    session['captcha'] = captcha_text
    session['captcha_created_at'] = time.time()
    # ... 生成图片

def check_captcha(user_input):
    stored = session.get('captcha', '')
    created_at = session.get('captcha_created_at', 0)
    if time.time() - created_at > 300:
        return False  # 过期
    return secrets.compare_digest(stored.lower(), user_input.lower())
```

#### 3.2.2 手机验证码

| 属性 | 值 | 说明 |
|------|-----|------|
| 长度 | 6 位 | 纯数字 |
| 有效期 | 300 秒（5分钟） | 超时需重新获取 |
| 使用次数 | 一次性 | 验证后立即从 Session 清除 |
| 手机号绑定 | 是 | 验证手机号与验证码匹配 |
| 发送频率 | 60 秒间隔 | 前端按钮倒计时限制 |

> 注：当前为演示环境，验证码直接返回给客户端。生产环境需接入短信服务商（阿里云、腾讯云等）。

### 3.3 请求频率限制

```python
# 内存存储的请求记录
request_limits = {}  # {ip: [(timestamp, count), ...]}

def rate_limit(max_requests=5, window=60):
    """频率限制装饰器"""
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            client_ip = request.remote_addr
            current_time = time.time()
            
            # 清理过期记录
            if client_ip in request_limits:
                request_limits[client_ip] = [
                    (ts, cnt) for ts, cnt in request_limits[client_ip]
                    if current_time - ts < window
                ]
            
            # 检查总请求次数
            total = sum(cnt for _, cnt in request_limits.get(client_ip, []))
            if total >= max_requests:
                return jsonify({'success': False, 
                              'message': '请求过于频繁'}), 429
            
            # 记录本次请求
            request_limits.setdefault(client_ip, [])
            request_limits[client_ip].append((current_time, 1))
            
            return f(*args, **kwargs)
        return decorated_function
    return decorator
```

| 接口 | 限制次数 | 时间窗口 | 触发响应 |
|------|----------|----------|----------|
| `/login` | 5 次 | 60 秒 | 429 Too Many Requests |

**局限性说明**：
- 基于内存存储，服务重启后记录丢失
- 多实例部署时不共享限制状态
- 生产环境建议改用 Redis 实现分布式限流

## 5. 数据安全防护

### 5.1 SQL 注入防护

**防护方式**：全部使用参数化查询（Prepared Statements），禁止字符串拼接 SQL。

```python
# ✅ 安全：参数化查询
conn.execute('SELECT * FROM users WHERE id = ?', (user_id,))

# ❌ 危险：字符串拼接（项目中不存在此类代码）
# conn.execute(f"SELECT * FROM users WHERE id = {user_id}")
```

**全系统参数化查询覆盖情况**：

| 模块 | 查询数量 | 是否全部参数化 |
|------|----------|---------------|
| 用户管理 | ~15 条 | ✅ 是 |
| 课程管理 | ~8 条 | ✅ 是 |
| 报名缴费 | ~6 条 | ✅ 是 |
| 签到管理 | ~10 条 | ✅ 是 |
| 消息中心 | ~6 条 | ✅ 是 |
| 教学管理 | ~12 条 | ✅ 是 |

### 5.2 XSS 防护

| 防护层 | 措施 | 说明 |
|--------|------|------|
| 模板层 | Jinja2 自动转义 | `{{ variable }}` 默认 HTML 转义 |
| 输入层 | 内容长度限制 | 消息内容限制 500 字符 |
| 输出层 | 不直接输出用户输入 | API 返回 JSON，不拼接 HTML |

**消息内容处理**：
```python
content = (request.form.get('content') or '').strip()
# 存储原始文本，渲染时由前端或模板自动转义
```

### 5.3 CSRF 防护

| 措施 | 实现方式 | 说明 |
|------|----------|------|
| SameSite Cookie | `SESSION_COOKIE_SAMESITE='Lax'` | 第三方 POST 不携带 Cookie |
| Session 绑定 | 服务端存储登录态 | 不依赖前端 Token |
| 敏感操作校验 | 已登录 + 权限检查 | 所有写操作都校验 Session |

> 注：当前未使用 CSRF Token 模式，依赖 SameSite Cookie 和 Session 校验。如需更高安全性，可在表单中增加 CSRF Token。

## 6. 业务安全设计

### 6.1 数据库事务安全

#### 6.1.1 事务使用场景

| 操作 | 事务范围 | 涉及表 |
|------|----------|--------|
| 缴费报名 | `BEGIN → INSERT enrollments → INSERT alternate_courses → UPDATE courses → COMMIT` | enrollments, alternate_courses, courses |
| 删除课程 | `BEGIN → DELETE alternate_courses → DELETE attendances → DELETE enrollments → DELETE courses → COMMIT` | 4 张表 |
| 调整学员 | `BEGIN → UPDATE courses(-1) → UPDATE courses(+1) → UPDATE enrollments → COMMIT` | courses, enrollments |
| 移除学员 | `BEGIN → UPDATE courses(-1) → DELETE attendances → DELETE enrollments → COMMIT` | courses, attendances, enrollments |
| 临时调课 | `BEGIN → INSERT attendances → UPDATE enrollments → COMMIT` | attendances, enrollments |
| 添加学员 | `BEGIN → INSERT users → INSERT enrollments → UPDATE courses → COMMIT` | users, enrollments, courses |
| 签到提交 | `BEGIN → INSERT/UPDATE attendances → UPDATE enrollments → COMMIT` | attendances, enrollments |

#### 6.1.2 事务失败处理

```python
try:
    conn.execute('BEGIN TRANSACTION')
    # ... 执行多条 SQL ...
    conn.commit()
except Exception as e:
    conn.rollback()
    return jsonify({'success': False, 'message': '操作失败，请重试'})
finally:
    conn.close()
```

### 6.2 并发安全

| 场景 | 风险 | 防护措施 |
|------|------|----------|
| 同时报名同一课程 | 超卖（人数超过 max_students） | 数据库事务 + 二次校验 |
| 同时调整学员 | 课程人数不一致 | 数据库事务 |
| 同时签到 | attendance_count 计算错误 | 单条记录 UPDATE |

**报名防超卖机制**：
```python
# 报名意向阶段（Session 级，非事务）
course = conn.execute('''
    SELECT * FROM courses 
    WHERE id = ? AND current_students < max_students
''', (course_id,)).fetchone()

# 缴费阶段（数据库事务内再次校验）
course = conn.execute('''
    SELECT * FROM courses 
    WHERE id = ? AND current_students < max_students
''', (course_id,)).fetchone()

existing = conn.execute('''
    SELECT * FROM enrollments 
    WHERE user_id = ? AND course_id = ?
''', (user_id, course_id)).fetchone()
```

## 8. 安全事件响应

### 8.1 异常处理策略

```python
@app.errorhandler(404)
def not_found(error):
    if request.is_json or request.path.startswith('/api/'):
        return jsonify({'success': False, 'message': '请求的资源不存在'}), 404
    return render_template('login.html', error='页面不存在'), 404

@app.errorhandler(500)
def internal_error(error):
    logging.error(f'服务器内部错误: {error}', exc_info=True)
    if request.is_json or request.path.startswith('/api/'):
        return jsonify({'success': False, 'message': '服务器内部错误'}), 500
    return render_template('login.html', error='服务器内部错误'), 500

@app.errorhandler(429)
def rate_limit_handler(error):
    if request.is_json or request.path.startswith('/api/'):
        return jsonify({'success': False, 'message': '请求过于频繁'}), 429
    return render_template('login.html', error='请求过于频繁'), 429
```

### 8.2 安全事件分级

| 级别 | 事件类型 | 响应措施 |
|------|----------|----------|
| 紧急 | 数据库泄露、大规模密码破解 | 立即下线、通知用户、强制重置密码 |
| 高危 | 管理员账号被盗、SQL 注入成功 | 冻结账号、审计日志、修复漏洞 |
| 中危 | 验证码被批量破解、暴力登录 | 加强限流、增加验证码复杂度 |
| 低危 | 异常访问模式、扫描行为 | 记录日志、加入黑名单 |
