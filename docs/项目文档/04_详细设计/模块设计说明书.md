## 1. 概述

### 1.1 编写目的
本文档从模块划分、接口定义、处理流程三个维度，对墨香书院书法报名系统进行详细设计，为编码实现和后续维护提供技术依据。

### 1.2 系统背景
- **系统名称**：墨香书院书法报名系统
- **开发框架**：Flask 2.x + Jinja2 + Flask-SocketIO
- **数据库**：SQLite（开发/测试）/ MySQL（生产）
- **部署方式**：Nginx + Gunicorn + Flask

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| Session | Flask 服务端会话，存储用户登录态 |
| PBKDF2 | 密码哈希算法（迭代 10 万次） |
| SocketIO | WebSocket 封装库，用于实时消息推送 |
| 签到 | 记录学员每节课的出勤状态（present/absent） |
| 备选课程 | 学员报名时指定的第二志愿课程 |

| 路由 | 方法 | 功能 | 访问控制 |
|------|------|------|----------|

#### 3.3.3 密码安全策略

```
注册/修改密码
    ↓
生成随机 salt（16 字节 hex）
    ↓
PBKDF2-HMAC-SHA256(password, salt, 100000 次迭代)
    ↓
存储格式: "salt$hash"
    ↓
验证时重新计算并 compare_digest 比对
```

#### 3.3.4 登录流程

```
[开始]
  ↓
判断登录方式 ──→ 手机号验证码 ──→ 验证验证码 → 查找/创建用户 → 写 Session → 跳转首页
  │
  └──→ 用户名密码 ──→ 验证验证码 → 验证密码 → 写 Session → 跳转首页
```

**Session 写入字段**：`user_id`, `username`, `name`, `role`, `profile_complete`

#### 3.7.1 模块职责
- 管理员按课程进行签到登记
- 自动计算课时数（attendance_count）
- 支持临时调课签到
- 课时统计分析（低于平均课时的学员标记）

#### 3.7.2 路由列表

| 路由 | 方法 | 功能 | 访问控制 |
|------|------|------|----------|
| `/admin/attendance/<course_id>` | GET/POST | 签到页面 / 提交签到 | 管理员 |
| `/admin/hours_management` | GET | 课时管理统计 | 管理员 |
| `/temp_adjust` | POST | 临时调课（AJAX） | 管理员 |

#### 3.7.3 签到状态变更逻辑

```
提交签到表单
  ↓
遍历该课程所有报名学员
  ↓
获取学员本次出勤状态（present/absent）
  ↓
若已有记录 → UPDATE attendances
若首次记录 → INSERT INTO attendances
  ↓
比对 old_status 与 new_status：
    old≠present 且 new=present → attendance_count + 1
    old=present 且 new≠present → attendance_count - 1
```

#### 3.7.4 课时管理统计逻辑

```
遍历所有课程
  ↓
获取该课程所有学员及 attendance_count
  ↓
计算平均课时 = SUM(attendance_count) / COUNT(students)
  ↓
标记 attendance_count < avg 的学员为 below_average
  ↓
计算缺口 hours_gap = avg - attendance_count（四舍五入）
```

#### 3.8.3 联系人排序算法

```sql
SELECT u.id, u.name,
    (子查询: 最后消息时间) AS last_message_time,
    (子查询: 最后消息内容) AS last_message,
    cp.created_at AS pinned_at
FROM users u
LEFT JOIN chat_pins cp ON cp.user_id = ? AND cp.pinned_user_id = u.id
WHERE u.role = 'user'
ORDER BY 
    (cp.created_at IS NOT NULL) DESC,  -- 置顶优先
    cp.created_at DESC,                 -- 置顶时间倒序
    last_message_time DESC              -- 最新消息时间倒序
```

#### 3.8.4 消息发送流程

```
用户发送消息
  ↓
校验接收者存在
  ↓
INSERT INTO messages (sender_id, receiver_id, content)
  ↓
返回 JSON 成功响应
  ↓
前端轮询 /api/conversation 拉取最新消息
```

```
输入: password (明文)
  ↓
salt = secrets.token_hex(16)          // 32字符随机盐
  ↓
key = PBKDF2-HMAC-SHA256(
    password=utf-8编码,
    salt=utf-8编码,
    iterations=100000
)
  ↓
存储: f"{salt}${key.hex()}"

验证:
  解析存储值 → salt, stored_key
  重新计算 key = PBKDF2(...)
  secrets.compare_digest(key.hex(), stored_key)
```

### 5.2 签到状态变更事务

```
输入: enrollment_id, class_date, new_status
  ↓
查询现有记录
  ↓
IF 存在记录:
    old_status = 记录.status
    UPDATE attendances SET status=new_status
ELSE:
    old_status = None
    INSERT INTO attendances (...)
  ↓
IF old_status != 'present' AND new_status == 'present':
    UPDATE enrollments SET attendance_count = attendance_count + 1
ELIF old_status == 'present' AND new_status != 'present':
    UPDATE enrollments SET attendance_count = attendance_count - 1
```

## 6. 安全性设计

### 6.1 认证安全
- Session Cookie 设置 `HttpOnly=True`, `SameSite=Lax`
- Session 过期时间 2 小时
- 验证码 5 分钟过期，使用后立即失效
- 登录接口 60 秒内限流 5 次

### 6.2 数据安全
- 密码使用 PBKDF2 10 万次迭代哈希
- 敏感操作（删除课程、移除学员）使用数据库事务
- 管理员操作接口统一校验 `session['role'] == 'admin'`

### 6.3 输入校验
- 手机号正则：`^1[3-9]\d{9}$`
- 姓名长度限制：1-50 字符
- 消息内容限制：不超过 500 字符
- 所有 POST 接口校验 CSRF（通过 Session 隐式保护）
