### 2.2 关键判断条件

| 判断节点 | 判断条件 | 失败处理 |
|----------|----------|----------|
| 验证码匹配 | `user_code == session['verification_code']` | 返回"验证码错误" |
| 验证码过期 | `current_time - created_at > 300秒` | 返回"验证码已过期" |
| 手机号重复 | `SELECT * FROM users WHERE phone = ?` | 返回"手机号已注册" |
| 姓名长度 | `1 <= len(name) <= 50` | 返回"姓名长度不符" |
| 手机号格式 | `^1[3-9]\d{9}$` | 返回"手机号格式不正确" |

### 2.3 数据变化

| 数据对象 | 变化 | 说明 |
|----------|------|------|
| `users` 表 | INSERT | 新增用户记录，role='user' |
| `session` | 写入 | `user_id`, `username`, `role`, `profile_complete=True` |
| `session` | 清除 | `verification_code`, `verification_code_created_at` |

### 3.2 密码验证算法流程

```
[输入: password, stored_hash]
        │
        ▼
┌─────────────────────────┐
│ stored_hash 包含 '$' ?  │
└───────────┬─────────────┘
            │
    ┌───────┴───────┐
    │ 是             │ 否（旧版格式）
    ▼               ▼
┌──────────┐  ┌──────────────┐
│ 按 $ 分割 │  │ SHA256(password)
│ salt, key │  │ 得到 old_hash
└────┬─────┘  └──────┬───────┘
     │               │
     ▼               ▼
┌──────────────────────────┐
│ PBKDF2-HMAC-SHA256       │
│ (password, salt, 100000) │
│ 得到 new_key              │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│ compare_digest(new_key,  │
│ key) 或 compare_digest(  │
│ old_hash, stored_hash)   │
└───────────┬──────────────┘
            │
    ┌───────┴───────┐
    │ True          │ False
    ▼               ▼
 [验证通过]      [验证失败]
```

### 3.3 登录方式对比

| 特性 | 用户名密码登录 | 手机号验证码登录 |
|------|--------------|----------------|
| 输入项 | 用户名 + 密码 + 图形验证码 | 手机号 + 6位短信验证码 |
| 用户不存在时 | 拒绝登录 | 自动创建新用户 |
| 密码处理 | 验证 PBKDF2 哈希 | 生成随机密码并哈希存储 |
| 信息完整性 | 按数据库状态判断 | 自动创建时 name='未知'，需完善 |
| 适用场景 | 已有账号的老用户 | 首次使用的新用户 |

### 4.2 事务内操作顺序

```
BEGIN TRANSACTION
    │
    ├── 1. 创建报名记录（enrollments）
    │      失败 → ROLLBACK
    │
    ├── 2. 创建备选课程记录（alternate_courses）
    │      失败 → ROLLBACK
    │      （每创建一个记录检查一次 course_id 是否存在）
    │
    └── 3. 更新课程人数（courses.current_students + 1）
           失败 → ROLLBACK
    │
COMMIT
```

### 4.3 关键判断条件

| 判断节点 | 条件 | 失败返回 |
|----------|------|----------|
| 用户登录 | `'user_id' in session` | 重定向 `/login` |
| 信息完整 | `name != '未知' AND child_name != '未知' AND phone != None` | "请先完善个人信息" |
| 课程存在 | `SELECT * FROM courses WHERE id=? AND current_students < max_students` | "课程不存在或已满" |
| 未重复报名 | `SELECT * FROM enrollments WHERE user_id=? AND course_id=?` | "您已报名该课程" |

### 6.2 管理员联系人排序流程

```
[输入: current_user_id]
        │
        ▼
┌─────────────────────────────────────┐
│ SELECT u.id, u.name,                │
│   (SELECT MAX(created_at)           │
│    FROM messages                    │
│    WHERE (sender_id=u.id AND        │
│           receiver_id=?)            │
│       OR (sender_id=? AND           │
│           receiver_id=u.id))        │
│   AS last_message_time,             │
│   (SELECT content FROM messages     │
│    WHERE ... ORDER BY created_at    │
│    DESC LIMIT 1) AS last_message,   │
│   cp.created_at AS pinned_at        │
│ FROM users u                        │
│ LEFT JOIN chat_pins cp ON           │
│   cp.user_id=? AND                  │
│   cp.pinned_user_id=u.id            │
│ WHERE u.role='user'                 │
│ ORDER BY                            │
│   (cp.created_at IS NOT NULL) DESC, │
│   cp.created_at DESC,               │
│   last_message_time DESC            │
└─────────────────────────────────────┘
```

### 8.2 级联删除顺序

```
删除顺序（必须按此顺序，否则外键约束失败）：

1. alternate_courses
   ├── 直接关联该 course_id 的记录
   └── 通过 enrollment_id 间接关联的记录

   └── 直接关联该 course_id 的记录

   └── 课程本身
```

### 9.2 课时统计流程

```
[管理员进入课时管理]
        │
        ▼
[后端查询所有课程]
        │
        ▼
[FOR course IN courses:]
        │
        ├── 查询该课程所有学员
        │   SELECT u.*, e.attendance_count
        │   FROM users u
        │   JOIN enrollments e ON u.id = e.user_id
        │   WHERE e.course_id = ?
        │
        ├── 计算平均课时
        │   avg = SUM(attendance_count) / COUNT(students)
        │
        └── FOR student IN students:
                │
                ├── IF attendance_count < avg:
                │     below_average = True
                │     hours_gap = round(avg - attendance_count)
                │   ELSE:
                │     below_average = False
                │     hours_gap = 0
                │
                └── 存入 course_students[course_id]
        │
        ▼
[渲染 hours_management.html]
```

### 9.3 置顶联系人流程

```
[管理员进入消息中心]
        │

        │

        │
        ├── 查询是否已置顶
        │   SELECT id FROM chat_pins
        │   WHERE user_id=? AND pinned_user_id=?
        │
        ├── IF 已存在:
        │     DELETE FROM chat_pins
        │     pinned = False
        │   ELSE:
        │     INSERT INTO chat_pins (user_id, pinned_user_id)
        │     pinned = True
        │
        └── 返回 {success: true, pinned: pinned}
        │
        ▼
[前端刷新联系人列表]
[置顶联系人移到列表顶部]
```
