#### 2.1.1 User（用户）

```python
class User:
    """用户实体类，对应 users 表"""
    
    # 属性
    id: int                    # 主键，自增
    username: str              # 登录用户名，唯一，非空
    password: str              # 密码哈希值（salt$hash 格式），非空
    name: str                  # 家长姓名
    phone: str                 # 手机号
    email: str                 # 邮箱
    child_name: str            # 孩子姓名
    role: str                  # 角色：'admin' 或 'user'，默认 'user'
    
    # 方法
    def is_profile_complete(self) -> bool:
        """判断个人信息是否完整"""
        return self.name != '未知' and self.child_name != '未知' and bool(self.phone)
    
    def is_admin(self) -> bool:
        """判断是否为管理员"""
        return self.role == 'admin'
```

### 2.2 服务类（Service Classes）

#### 2.2.1 DatabaseConnection（数据库连接上下文管理器）

```python
class DatabaseConnection:
    """数据库连接上下文管理器
    
    职责：
    1. 自动获取数据库连接（SQLite 或 MySQL）
    2. 退出时自动提交或回滚事务
    3. 自动关闭连接
    
    用法：
        with DatabaseConnection() as conn:
            cursor = conn.execute("SELECT * FROM users")
            rows = cursor.fetchall()
    """
    
    # 属性
    conn: Connection           # 数据库连接对象
    
    # 方法
    def __enter__(self) -> Connection:
        """进入上下文时获取连接"""
        self.conn = get_db_connection()
        return self.conn
    
    def __exit__(self, exc_type, exc_val, exc_tb) -> None:
        """退出上下文时处理事务并关闭连接"""
        if exc_type:
            self.conn.rollback()
        else:
            self.conn.commit()
        self.conn.close()
```

## 3. 接口设计

### 3.1 接口设计规范

**通用响应格式**：

```json
// 成功响应
{
    "success": true,
    "message": "操作成功",
    "data": { ... }          // 可选，视接口而定
}

// 失败响应
{
    "success": false,
    "message": "错误描述"
}
```

**认证方式**：基于 Flask Session Cookie（服务端会话）

**错误状态码**：

| 状态码 | 含义 | 触发场景 |
|--------|------|----------|
| 200 | OK | 请求成功 |
| 401 | Unauthorized | 未登录 |
| 403 | Forbidden | 无权限（非管理员） |
| 404 | Not Found | 资源不存在 |
| 429 | Too Many Requests | 请求频率超限 |
| 500 | Internal Server Error | 服务器内部错误 |

### 3.2 用户认证接口

#### 3.2.1 用户注册

```
POST /register
```

**请求参数（Form Data）**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| phone | string | 是 | 手机号，11位数字 |
| verification_code | string | 是 | 6位数字验证码 |
| name | string | 是 | 家长姓名，1-50字符 |
| child_name | string | 是 | 孩子姓名，1-50字符 |

**响应**：

```
HTTP/1.1 302 Found
Location: /register_success

// 或在验证失败时
HTTP/1.1 200 OK
Content-Type: text/html
// 渲染 register.html，带 error 提示
```

**处理流程**：
1. 校验验证码（Session 中存储的 6 位码，5 分钟有效）
2. 校验手机号格式
3. 检查手机号是否已注册
4. 生成随机用户名 `user_xxxxxxxx` 和随机密码

### A.3 WebSocket 连接示例

```javascript
// 前端建立 WebSocket 连接
const socket = io('ws://localhost:5000');

// 连接成功
socket.on('connected', (data) => {
    console.log(data.message);  // "欢迎 张三 连接成功！"
});

// 加入课程房间
socket.emit('join_course', {course_id: 1});
socket.on('joined_course', (data) => {
    console.log(data.message);  // "已加入课程 1 的房间"
});

// 发送消息到房间
socket.emit('send_message', {
    message: '大家好',
    room: 'course_1'
});

// 接收广播消息
socket.on('new_message', (data) => {
    console.log(`${data.username}: ${data.message}`);
});
```
