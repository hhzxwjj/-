## 2. 模块详细说明

### 2.1 系统框架模块

**模块编号**：M1  
**模块名称**：系统框架模块  
**源文件**：`app/__init__.py`  

#### 2.1.1 职责描述
负责 Flask Web 应用和 SocketIO 实时通信服务的初始化，配置安全参数，注册静态资源和模板路径。

#### 2.1.2 子模块分解

| 子模块编号 | 子模块名称 | 职责 | 关键代码 |
|-----------|-----------|------|----------|
| M1-1 | Flask 应用初始化 | 创建 Flask 实例，配置 SECRET_KEY、Session Cookie 安全属性 | `app = Flask(__name__)` |
| M1-2 | SocketIO 初始化 | 创建 WebSocket 服务实例，绑定到 Flask 应用 | `socketio = SocketIO(app)` |
| M1-3 | 静态资源配置 | 设置 static_folder 和 template_folder 路径 | `app.static_folder = ...` |

#### 2.1.3 接口定义

| 接口名 | 类型 | 说明 |
|--------|------|------|
| `app` | Flask 实例 | 全局 Flask 应用对象 |
| `socketio` | SocketIO 实例 | 全局 WebSocket 服务对象 |

#### 2.1.4 配置参数

| 配置项 | 值 | 说明 |
|--------|-----|------|
| SECRET_KEY | 环境变量或随机 64 位 hex | Session 加密密钥 |
| SESSION_COOKIE_SECURE | False（开发）/ True（生产） | HTTPS 传输 |
| SESSION_COOKIE_HTTPONLY | True | 禁止 JS 访问 Cookie |
| SESSION_COOKIE_SAMESITE | Lax | CSRF 防护 |
| PERMANENT_SESSION_LIFETIME | 2 小时 | Session 过期时间 |

### 2.2 数据访问模块

**模块编号**：M2  
**模块名称**：数据访问模块  
**源文件**：`app/models/database.py`  

#### 2.2.1 职责描述
统一管理数据库连接，支持 SQLite（开发/测试）和 MySQL（生产）两种模式，提供连接上下文管理器自动处理事务，负责数据表结构的初始化创建。

#### 2.2.2 子模块分解

| 子模块编号 | 子模块名称 | 职责 | 关键函数/类 |
|-----------|-----------|------|------------|
| M2-1 | 数据库连接工厂 | 根据环境配置返回对应数据库连接 | `get_db_connection()` |
| M2-2 | 数据库连接上下文管理器 | 自动提交/回滚事务，自动关闭连接 | `DatabaseConnection` 类 |
| M2-3 | 数据表初始化 | 创建 7 张数据表，启用外键约束 | `init_db()` |
| M2-4 | 双数据库切换 | 通过 `USE_MYSQL` 环境变量切换数据库类型 | `_get_mysql_conn()` / `_get_sqlite_conn()` |

#### 2.2.3 数据表清单

| 表名 | 用途 | 记录数（预估） | 关联表 |
|------|------|---------------|--------|
| `users` | 用户信息 | 100~500 | — |
| `courses` | 课程信息 | 5~20 | — |
| `enrollments` | 报名记录 | 200~1000 | users, courses |
| `attendances` | 签到记录 | 2000~10000 | enrollments |
| `alternate_courses` | 备选课程 | 100~500 | enrollments, courses |
| `messages` | 私聊消息 | 5000~50000 | users |
| `chat_pins` | 置顶联系人 | 50~200 | users |

#### 2.2.4 接口定义

```python
# 获取数据库连接
def get_db_connection() -> sqlite3.Connection | pymysql.Connection

# 数据库连接上下文管理器
class DatabaseConnection:
    def __enter__(self) -> Connection
    def __exit__(self, exc_type, exc_val, exc_tb) -> None

# 初始化数据库（仅 SQLite 模式）
def init_db() -> None
```

### 2.3 用户管理模块

**模块编号**：M3  
**模块名称**：用户管理模块  
**源文件**：`app/routes/main.py`（第 27~584 行）  

#### 2.3.1 职责描述
负责用户的全生命周期管理：注册、登录、登出、个人信息维护、密码修改。同时提供验证码生成与验证服务，以及默认管理员账号的初始化。

#### 2.3.2 子模块分解

| 子模块编号 | 子模块名称 | 职责 | 关键函数 |
|-----------|-----------|------|----------|
| M3-1 | 用户注册 | 手机号验证码注册，自动生成用户名密码 | `register()` |
| M3-2 | 用户登录 | 支持用户名密码和手机号验证码双模式 | `login()` |
| M3-3 | 个人信息管理 | 完善资料、更新资料、用户中心展示 | `complete_profile()`, `update_profile()`, `user_center()` |
| M3-4 | 密码管理 | 修改密码（需验证原密码） | `change_password()` |
| M3-5 | 验证码服务 | 生成 4 位数字验证码图片，验证用户输入 | `generate_captcha()`, `check_captcha()` |
| M3-6 | 管理员初始化 | 系统启动时自动创建 admin 账号 | `init_admin()` |

#### 2.3.3 用户信息完整性检查中间件

```python
@app.before_request
def check_profile_completeness():

                      'captcha', 'static', 'complete_profile', 
                      'update_profile', 'logout']
```

### 2.4 课程与报名模块

**模块编号**：M4  
**模块名称**：课程与报名模块  
**源文件**：`app/routes/main.py`（第 606~807 行）  

#### 2.4.1 职责描述
负责课程的展示与管理、学员的报名意向提交、模拟缴费流程处理，以及备选课程的记录管理。

#### 2.4.2 子模块分解

| 子模块编号 | 子模块名称 | 职责 | 关键函数 |
|-----------|-----------|------|----------|
| M4-1 | 课程管理 | 课程列表展示、课程详情查询、管理员增删课程 | `courses()`, `course_detail()`, `add_course()`, `delete_course()` |
| M4-2 | 报名意向 | 学员提交报名意向，校验余量和重复报名 | `enroll()` |
| M4-3 | 缴费处理 | 模拟缴费，创建报名记录和备选课程记录 | `payment()` |
| M4-4 | 备选课程 | 报名时指定备选课程，缴费时创建备选记录 | —（内嵌于 payment） |

#### 2.4.3 报名缴费状态流转

```
未报名
  ↓ 点击"立即报名"
已选课程（Session 保存 course_id）
  ↓ 点击"确认缴费"
已报名（enrollments.status='paid'）
  ↓ 管理员删除课程 / 管理员移除学员
已取消
```

### 2.6 消息与通信模块

**模块编号**：M6  
**模块名称**：消息与通信模块  
**源文件**：`app/routes/main.py`（第 1003~1195 行）  

#### 2.6.1 职责描述
实现管理员与学员之间的私聊功能，包括消息发送、聊天记录查询、联系人列表管理（置顶排序），以及基于 WebSocket 的实时通信。

#### 2.6.2 子模块分解

| 子模块编号 | 子模块名称 | 职责 | 关键函数 |
|-----------|-----------|------|----------|
| M6-1 | 私聊消息 | 发送消息、获取聊天记录、标记已读 | `send_message()`, `api_conversation()` |
| M6-2 | 联系人管理 | 获取联系人列表、置顶/取消置顶 | `api_contacts()`, `api_toggle_pin()` |
| M6-3 | WebSocket 实时通信 | 建立长连接、管理房间、广播消息 | `handle_connect()`, `handle_send_message()` 等 |
| M6-4 | 消息通知 | 签到更新通知、报名通知、调课通知 | `handle_attendance_update()`, `handle_course_enrollment()` |

#### 2.6.3 联系人排序规则

```
1. 置顶优先（cp.created_at IS NOT NULL DESC）
2. 置顶时间倒序（cp.created_at DESC）
3. 最后消息时间倒序（last_message_time DESC）
```

| 课程房间 | `course_1` | 课程相关人员 | 签到更新、课程动态 |

```python
    # 60 秒内最多 5 次请求
```
