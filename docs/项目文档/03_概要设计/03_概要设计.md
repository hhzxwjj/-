## 3.4 体系结构设计

### 系统架构图

本项目采用 **单体 MVC 架构**（Model-View-Controller），前后端通过模板渲染方式交互，实时通信通过 WebSocket 实现。

```
┌─────────────────────────────────────────────────────────────┐
│                     表示层 (View)                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   Jinja2    │  │   HTML5     │  │  JavaScript │         │
│  │   模板引擎   │  │   页面结构   │  │  交互逻辑   │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                              │
│  模板文件: login.html, courses.html, admin.html, ...         │
│  静态资源: style.css, script.js                              │
└─────────────────────────────────────────────────────────────┘
                              │ 请求/响应 (HTTP)
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   控制层 (Controller)                        │
│                    Flask 路由处理                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  app/routes/main.py                                   │  │
│  │  • 用户认证路由 (login, register, logout)             │  │
│  │  • 课程路由 (courses, course_detail)                  │  │
│  │  • 报名路由 (enroll, payment)                         │  │
│  │  • 管理路由 (admin, attendance, hours_management)     │  │
│  │  • 消息路由 (messages, send_message)                  │  │
│  │  • WebSocket 事件处理 (connect, send_message, ...)    │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │ 数据操作
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     模型层 (Model)                           │
│  ┌────────────────────────┐  ┌──────────────────────────┐  │
│  │  app/models/database.py │  │   app/routes/main.py     │  │
│  │  • 数据库初始化         │  │   • 业务逻辑处理          │  │
│  │  • 表结构定义           │  │   • 数据校验              │  │
│  │  • 连接管理             │  │   • 事务控制              │  │
│  └────────────────────────┘  └──────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    数据层 (Data)                             │
│                 SQLite 关系型数据库                          │
│              calligraphy_system.db 文件                      │
└─────────────────────────────────────────────────────────────┘
```

### 架构特点

- **轻量级**：单进程 Flask 应用，无需额外中间件，适合教学演示
- **模板渲染**：后端直接渲染 HTML，降低前端开发复杂度
- **实时通信**：SocketIO 提供 WebSocket 支持，实现消息实时推送
- **分层清晰**：模型-视图-控制器职责分离，便于维护扩展

### 关键类/函数说明

| 名称 | 类型 | 职责 |
|------|------|------|
| `DatabaseConnection` | 上下文管理器 | 封装数据库连接的获取与释放，自动处理 commit/rollback |
| `hash_password` | 工具函数 | 使用 PBKDF2-HMAC-SHA256 对密码进行哈希 |
| `verify_password` | 工具函数 | 验证密码与哈希值是否匹配，兼容旧版 SHA256 |
| `rate_limit` | 装饰器 | 基于 IP 的请求频率限制，防止暴力破解 |
| `generate_captcha` | 工具函数 | 生成随机验证码图片（Base64 或二进制流） |
| `check_captcha` | 工具函数 | 验证用户输入的验证码，5分钟过期 |
| `check_profile_completeness` | 中间件 | 检查用户个人信息是否完整，不完整则强制跳转 |

### 3.9.3 物理设计（建表语句）

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    name TEXT,
    phone TEXT,
    email TEXT,
    child_name TEXT,
    role TEXT DEFAULT 'user'
);

-- 课程表
CREATE TABLE IF NOT EXISTS courses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT,
    total_hours INTEGER,
    price REAL,
    start_time TEXT,
    end_time TEXT,
    max_students INTEGER,
    current_students INTEGER DEFAULT 0
);

-- 报名记录表
CREATE TABLE IF NOT EXISTS enrollments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    course_id INTEGER,
    status TEXT DEFAULT 'pending',
    payment_amount REAL,
    payment_time TEXT,
    attendance_count INTEGER DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES users (id),
    FOREIGN KEY (course_id) REFERENCES courses (id)
);

-- 签到记录表
CREATE TABLE IF NOT EXISTS attendances (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    enrollment_id INTEGER,
    class_date TEXT,
    class_number INTEGER,
    status TEXT DEFAULT 'absent',
    FOREIGN KEY (enrollment_id) REFERENCES enrollments (id)
);

-- 备选课程表
CREATE TABLE IF NOT EXISTS alternate_courses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    enrollment_id INTEGER,
    course_id INTEGER,
    status TEXT DEFAULT 'pending',
    FOREIGN KEY (enrollment_id) REFERENCES enrollments (id),
    FOREIGN KEY (course_id) REFERENCES courses (id)
);

-- 消息表
CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sender_id INTEGER,
    receiver_id INTEGER,
    content TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    status TEXT DEFAULT 'unread',
    FOREIGN KEY (sender_id) REFERENCES users (id),
    FOREIGN KEY (receiver_id) REFERENCES users (id)
);
```

| idx_attendances_enrollment | attendances | enrollment_id | - | 查询报名记录的签到 |
| idx_messages_sender | messages | sender_id | - | 查询发送的消息 |
