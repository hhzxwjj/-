# 书法培训班管理系统

## 项目简介

书法培训班管理系统是一个基于 Flask 框架开发的 Web 应用，用于管理书法培训班的日常运营，包括学员管理、课程管理、签到管理、消息通知等功能。

## 功能特性

### 1. 用户管理
- **注册登录**：支持手机号验证码登录，新用户自动注册
- **个人信息**：完善和更新个人信息
- **密码修改**：支持修改登录密码
- **角色管理**：区分管理员和普通用户

### 2. 课程管理
- **课程列表**：查看所有课程信息
- **课程详情**：查看课程详细信息
- **课程报名**：支持学员报名课程
- **缴费流程**：模拟缴费验证过程

### 3. 签到管理
- **管理员签到**：管理员为学员进行签到
- **课时统计**：统计学员的签到次数和课时
- **临时调课**：支持学员临时调整课程
- **课时分析**：分析学员课时情况，标记低于平均值的学员

### 4. 消息通知
- **消息中心**：查看系统消息
- **实时通知**：通过 WebSocket 实现实时消息通知
- **管理员消息**：管理员向学员发送消息

### 5. 管理员功能
- **课程管理**：添加和管理课程
- **学生管理**：添加、移除和调整学生
- **数据统计**：查看系统数据统计和分析

## 核心功能使用

### 1. 学员报名流程
1. 访问登录页面，切换到"验证码登录"，输入手机号点击"获取验证码"
2. 验证码会显示在按钮下方，输入验证码登录（新用户自动注册）
3. 完善个人信息（首次登录会提示）
4. 浏览课程列表，选择感兴趣的课程
5. 点击报名按钮，完成缴费流程
6. 报名成功，开始学习

### 2. 管理员签到流程
1. 使用管理员账户登录
2. 进入管理员后台
3. 选择课程，点击"签到"按钮
4. 选择日期，为学员进行签到（出席/缺席）
5. 系统自动统计课时

### 3. 课时管理
1. 管理员登录后，进入"课时管理"页面
2. 查看各课程的学员课时情况
3. 系统会自动标记课时低于平均值的学员
4. 可以为学员安排临时调课

## 注意事项

1. **数据库**：系统使用 SQLite 数据库，首次启动时会自动创建
2. **验证码**：开发环境下手机验证码会直接显示在页面上，暂未接入真实短信服务商
3. **安全性**：本系统为演示版本，实际部署时需要加强安全措施
4. **性能**：适合小型培训班使用，大型培训班建议使用更强大的数据库

## 代码检查与安全说明（2026-05 更新）

基于当前代码实现，已补充以下说明，便于后续维护与部署：

### 已发现的主要风险点

1. **开发环境验证码直接返回**
   - `/send_verification_code` 在开发环境会直接返回短信验证码，便于测试，但若误用于生产环境，会造成验证码泄露风险。

2. **默认管理员弱口令风险**
   - 系统默认创建 `admin / admin123` 管理员账户。虽然文档中已注明，但实际部署时必须立即修改，否则存在高风险。

3. **WebSocket 房间鉴权仍需加强**
   - 部分 WebSocket 事件仅依赖 session 登录态，房间加入和消息广播的权限校验还可以进一步细化，避免越权订阅和跨房间消息误发。

4. **登录与验证码逻辑偏开发友好**
   - 手机号验证码登录、自动注册、生成临时密码等逻辑便于演示，但在正式环境需要增加短信服务、验证码风控和失败次数限制。

5. **请求频率限制能力有限**
   - 当前频率限制主要基于内存字典和客户端 IP，适合单机开发；如果多进程或多实例部署，限流效果会下降。

6. **日志与调试信息需分环境控制**
   - 当前代码存在调试输出、错误堆栈和验证码日志等信息，生产环境应避免记录敏感数据。

### 建议的安全加固措施

- 生产环境关闭 `debug=True`
- 将 `SESSION_COOKIE_SECURE` 在 HTTPS 环境下设为 `True`
- 接入真实短信验证码服务，并增加验证码有效期、失败次数与设备/IP 限制
- 为管理员账户强制首次登录修改密码
- 为消息接口、调课接口、签到接口补充更严格的授权校验与参数校验
- 将限流从内存字典迁移到 Redis 等共享存储
- 对所有用户输入进行长度、类型与业务合法性校验
