### 9.3.2 建模工具使用流程

**PlantUML + Draw.io 联合建模流程**：

1. **编写 PlantUML 代码**：使用 UML 2.5 标准语法在文本编辑器（VS Code + PlantUML 插件）中编写建模代码（`.puml` 文件）
2. **本地渲染验证**：通过 PlantUML Server 或本地 JAR 包渲染 PNG/SVG，确认语法正确
3. **导入 Draw.io 精调**：打开 https://app.diagrams.net/ → Arrange → Insert → Advanced → PlantUML → 粘贴代码 → 自动生成矢量图形
4. **调整样式布局**：在 Draw.io 中拖拽调整元素位置、配色、字体，统一项目视觉风格
5. **导出交付物**：导出为 PNG/SVG/PDF，放入 `docs/diagrams/` 作为最终证据；同时保留 `.drawio` 源文件供后续修改

**自动化生成脚本**：

项目根目录提供 `docs/diagrams/generate_drawio.py` 和 `generate_flow_drawio.py` 脚本，可直接生成符合 UML 2.5 规范的 `.drawio` 文件，避免手工绘制误差：

```bash
# 生成类图、组件图、部署图等结构图
python docs/diagrams/generate_drawio.py

# 生成活动图、流程图等动态图
python docs/diagrams/generate_flow_drawio.py
```
