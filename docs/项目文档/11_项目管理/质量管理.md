## 三、代码质量管理

### 3.1 代码规范

本项目遵循 `docs/代码规范.md` 中定义的编码标准，核心规范如下：

| 规范项 | 要求 | 示例 |
|--------|------|------|
| 命名规范 | 小写下划线命名法 | `get_db_connection`、`hash_password` |
| 函数注释 | 所有函数必须有Docstring | `"""使用PBKDF2对密码进行哈希加密"""` |
| 行长度 | 每行不超过120字符 | 超出时换行 |
| 缩进 | 4个空格 | 不使用Tab |
| 导入顺序 | 标准库→第三方库→本地模块 | `import os` → `from flask import ...` → `from app import ...` |
| 敏感信息 | 禁止硬编码密码/密钥 | 从环境变量读取 `SECRET_KEY`、`ADMIN_DEFAULT_PASSWORD` |

### 3.2 代码审查（Code Review）

| 审查项 | 检查内容 | 审查方式 |
|--------|----------|----------|
| 安全性 | SQL注入防护、XSS防护、密码加密 | 安全测试用例（37个） |
| 事务一致性 | 报名缴费、人员调整、删除课程 | 数据库验证 + pytest断言 |
| 异常处理 | 所有路由是否有错误处理 | `@app.errorhandler` + try/except |
| 权限控制 | 管理员接口是否校验role | pytest权限测试（10个场景） |
| 代码重复 | 函数复用情况 | 人工审查 |

### 3.3 静态代码检查
