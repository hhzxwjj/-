| 段位 | 说明 | 触发条件 |

| 主版本（X.0.0） | 重大架构变更 | 技术栈更换、架构重构 |

## 五、文档命名规范

### 5.1 Markdown文件命名

```
{编号}_文档名称.md          ← 阶段文档（如 01_可行性分析与软件开发计划.md）
{类型}_文档名称.md          ← 设计文档（如 体系结构_类设计_接口设计.md）
文档名称.md                 ← 独立文档（如 用户手册.md）
```

### 5.2 Word文件命名

```
墨香书院_文档名称.docx       ← 中文命名，与md文件对应（旧版）
{编号}_文档名称.docx         ← 阶段文档Word版本
文档名称.docx                ← 独立文档Word版本
```

### 5.3 建模图命名

```
{类型}_名称.drawio           ← 如 活动图_注册.drawio
```

### 8.2 文档更新流程

```
发现问题/需求变更 → 修改md源文件 → 更新版本号 → 重新生成docx → 确认子目录位置 → Git提交 → 通知相关人员
```

### 8.3 文档废弃流程

```
标记废弃 → 在文档头部添加废弃声明 → 保留历史版本 → 通知相关人员 → 从清单中移除
```
